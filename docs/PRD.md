# Winote - 태블릿 필기 앱 기획서 (PRD)

> **S펜/스타일러스 최적화 필기 앱** - Windows + Android 크로스플랫폼
>
> 버전: v2.0 (재미나이 피드백 반영)
> 작성일: 2026-01-12
> 상태: 기획 단계

---

## 1. 제품 정의

### 1.1 한 줄 정의
**"종이 같은 필기감 + 강력한 정리/검색 + 기기간 즉시 동기화"** 를 제공하는 S펜/스타일러스 최적화 노트 앱

### 1.2 제품명
- **Winote** (Windows + Note, Write + Note)

### 1.3 핵심 가치 (차별점)

| 가치 | 설명 | 기존 앱과의 차이 |
|------|------|-----------------|
| **펜이 주인공** | 저지연 입력 (<10ms), 손바닥 거부, 펜 버튼/제스처 지원 | 삼성노트보다 빠른 반응, GoodNotes급 필기감 |
| **PDF 워크플로** | 자료 열고 → 바로 쓰고 → 요약/태그 → 공유 | Notability 수준의 PDF 주석 기능 |
| **정리 자동화** | 제목/태그 자동 추천, 필기 검색 (OCR) | Notion급 정리 + 필기 특화 |
| **폴더 기반 동기화** | WinoteData 폴더를 OneDrive/Google Drive에 배치하면 자동 동기화 | Obsidian 방식의 심플한 동기화 |

| **크리에이터** | 아이디어 스케치, 브레인스토밍 | 무한 캔버스 + 다양한 펜/브러시 |


## 3. 기능 범위

### 3.1 Phase 0: PoC (필기감 검증) - 2~4일

> **핵심 목표**: 빈 화면에서 펜 입력 → 저지연 렌더링(압력/기울기) → 체감 테스트

| 기능 | 상세 | KPI |
|------|------|-----|
| 펜 입력 파이프라인 | Input → Resample → Filter → Render | 지연 <10ms (체감) |
| 압력/기울기 반영 | 필압에 따른 굵기, 기울기에 따른 음영 | 60/120Hz에서 끊김 없음 |
| 손바닥 거부 | 펜 입력 시 터치 무시 | 100% 정확도 |
| Android/Windows 테스트 | 양 플랫폼에서 동일 필기감 | 체감 지연 차이 없음 |

**Phase 0 통과 조건:**
- [ ] 60Hz 디스플레이에서 펜 끝과 선 사이 지연이 거의 없음
- [ ] 120Hz 디스플레이에서 더욱 부드러운 필기감
- [ ] 손바닥을 대고 필기해도 오작동 없음
- [ ] Windows Ink / Android S Pen 모두 정상 동작

## 5. 데이터 모델

### 5.1 핵심 엔티티

```typescript
// 노트
interface Note {
  id: string;
  folderId: string;
  title: string;
  tags: string[];
  isPinned: boolean;
  isFavorite: boolean;
  thumbnailPath: string;
  createdAt: Date;
  updatedAt: Date;
  lastOpenedAt: Date;
  deviceLastEdited: string;
  version: number;           // 동기화용 버전
  checksum: string;          // 변경 감지용
}

// 페이지
interface Page {
  id: string;
  noteId: string;
  order: number;
  templateType: 'blank' | 'lined' | 'grid' | 'dotted' | 'cornell';
  canvasType: 'infinite' | 'fixed';
  width: number;
  height: number;
  backgroundColor: string;
  strokeDataPath: string;    // 바이너리 스트로크 파일 경로
  createdAt: Date;
  updatedAt: Date;
}

// 스트로크 (바이너리 직렬화용)
interface Stroke {
  id: string;
  toolType: 'pen' | 'pencil' | 'marker' | 'highlighter';
  color: number;             // ARGB 정수
  baseWidth: number;
  opacity: number;
  points: StrokePoint[];
  boundingBox: BoundingBox;  // 공간 인덱싱용
  timestamp: number;
}

interface StrokePoint {
  x: number;                 // float16으로 저장 가능
  y: number;
  pressure: number;          // 0.0 ~ 1.0
  tilt: number;
  timestamp: number;
}

interface BoundingBox {
  minX: number;
  minY: number;
  maxX: number;
  maxY: number;
}
```

### 5.2 저장 구조 (개선됨)

> **핵심 변경**: 스트로크는 **바이너리 파일**로 저장, 메타데이터만 JSON/SQLite

```
📁 WinoteData/                          # 사용자 지정 위치 (OneDrive 등)
├── 📁 notes/
│   └── 📁 {noteId}/
│       ├── note.json                   # 노트 메타데이터 (JSON)
│       ├── thumbnail.png               # 썸네일
│       └── 📁 pages/
│           ├── page_001.meta.json      # 페이지 메타데이터
│           ├── page_001.strokes.bin    # 스트로크 바이너리 (Protobuf/FlatBuffers)
│           ├── page_001.cache.png      # 렌더 캐시 이미지 (선택)
│           ├── page_002.meta.json
│           ├── page_002.strokes.bin
│           └── ...
│
├── 📁 attachments/
│   └── 📁 {noteId}/
│       ├── {attachmentId}.pdf
│       └── ...
│
├── 📁 folders/
│   └── structure.json                  # 폴더 트리 구조
│
├── 📁 sync/
│   ├── local_state.json                # 로컬 동기화 상태
│   ├── pending_changes.json            # 동기화 대기 변경사항
│   └── conflicts/                      # 충돌 백업
│       └── {noteId}_{timestamp}/
│
├── settings.json                       # 앱 설정
└── index.sqlite                        # 검색/인덱스용 DB (로컬 캐시)
```

### 5.3 스트로크 바이너리 포맷

```
// page_001.strokes.bin 구조
┌──────────────────────────────────────────┐
│ Header (16 bytes)                        │
│ - Magic: "WINK" (4 bytes)               │
│ - Version: uint16                        │
│ - StrokeCount: uint32                    │
│ - Reserved: 6 bytes                      │
├──────────────────────────────────────────┤
│ Stroke Index Table                       │
│ - 각 스트로크의 오프셋/크기              │
├──────────────────────────────────────────┤
│ Stroke Data                              │
│ - StrokeHeader: toolType, color, width  │
│ - BoundingBox: minX, minY, maxX, maxY   │
│ - PointCount: uint32                     │
│ - Points[]: x, y, pressure, tilt, time  │
│   (각각 float16 또는 적절한 압축)        │
├──────────────────────────────────────────┤
│ Spatial Index (QuadTree 직렬화)          │
│ - 뷰포트 쿼리용 공간 인덱스              │
└──────────────────────────────────────────┘
```

**바이너리 저장의 장점:**
- JSON 대비 **5~10배 작은 파일 크기**
- 파싱 없이 **직접 메모리 매핑** 가능
- 부분 로드 가능 (스트로크 인덱스 테이블 활용)

| **파일** | path_provider | 파일 경로 관리 |
| **바이너리** | protobuf / flatbuffers | 스트로크 직렬화 |
| **PDF** | syncfusion_flutter_pdfviewer | PDF 렌더링 |
| **파일 감시** | watcher | 동기화 폴더 변경 감지 |
| **직렬화** | json_serializable | JSON 변환 |
| **테스트** | mocktail | 목 객체 |
