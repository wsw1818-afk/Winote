# 에러 대응 규칙

## 기본 흐름
실패 로그 원문 인용 → 원인 가설 → 최소 수정안 → 재검증 계획 → 재시도

## 같은 실패 2회 이상
대체 경로/우회 전략을 제시한다.

## 유형별 체크리스트

### Network/Permission
- 자격증명(.env), 권한/토큰, 방화벽·프록시
- 엔드포인트/HTTP 메서드, CORS(웹)
- 조치: 재시도(지수 백오프), 타임아웃·에러 핸들링 강화

### Build 실패
- Node/패키지 버전, 락파일 불일치, 중복/순환 의존
- 조치: 버전 고정, 충돌 패키지 교체, `--frozen-lockfile` 재설치

### Test 실패
- 환경변수 누락, Mock/Fixture, 시간·랜덤 의존성
- 조치: 테스트 격리/시드 고정, 경계값 추가, 타임아웃 조정

## 로그 분석 규칙
- 로그를 받으면 전체를 꼼꼼히 읽고 결론
- 일부만 보고 "됩니다!"라고 단정 금지
- RESULTS에 핵심 에러 원문 일부 포함 (민감정보는 마스킹)
